## 2. Улучшение работы агента

### 2.1 Чеклист перед правкой формул
- [ ] Проверить, что `price` — это цена купленной доли (YES или NO)
- [ ] Для NO не использовать `1/(1-price)`
- [ ] Прогнать тестовый кейс: NO, bet=30, price=0.27 → pnl=81.11 при победе

### 2.2 Чеклист при добавлении MANUAL_RESULTS
- [ ] Ключ — точное совпадение с текстом в Excel (колонка 3)
- [ ] outcome: "Yes" | "No" (с большой буквы)
- [ ] Для будущих матчей: outcome=None, в note — инструкция заполнить

### 2.3 Проверка после изменений
- Запустить `refresh.py`
- Прогнать скрипт проверки P&L (из чата) для всех закрытых ставок
- Сверить Итоговый P&L с ожидаемым

### 2.4 Не придумывать данные
- Если результат матча неизвестен — писать «нет данных», не вносить в MANUAL_RESULTS
- Если API недоступен — использовать кэш, не симулировать ответы

### 2.5 Использование плана
- Перед крупными изменениями читать `.cursor/plans/polymarket_analytics_agent_*.plan.md`
- Учитывать статус todo (completed/pending) при добавлении новых фич
