# Стресс-тесты и цель «$1M депозит»

## Что запускается

```bash
pytest tests/test_stress_formulas.py tests/test_goal_stress.py -v
pytest tests/   # полный набор (85+ тестов)
```

## Смысл проверок под цель роста капитала

1. **Формулы (test_stress_formulas)** — P&L, Kelly, ничьи, границы цен; без этого любая стратегия «до миллиона» математически нестабильна.
2. **Монте-Карло (test_goal_stress)** — упрощённый банкролл: `compute_kelly` + `calc_pnl`, случайные исходы с заданным `true_prob`:
   - при **положительном edge** медианный банкролл растёт;
   - при **отрицательном** — сжимается.
3. **Капы (test_goal_01, test_goal_04)** — при `MAX_BET_USD=50` доля капитала на одну ставку на больших банкроллах **не растёт**; путь к $1M — **очень много** качественных ставок и/или **пересмотр лимитов** после доказанного edge.
4. **Риск** — дневной стоп и порог slippage (`>` max, не `>=`) согласованы с тестами.
5. **Резолв рынков** — вспомогательные функции `gamma_resolve` не ломаются на типичных Yes/No.

## Вывод для цели $1M

- Проект **не обещает** достижение $1M: тесты проверяют **внутреннюю согласованность** и **качественное направление** при наличии реального edge.
- При текущих настройках **малый фиксированный макс. размер ставки** делает рост **медленным** в долларовом выражении; масштабирование — отдельное решение (лимиты, банкролл, законодательство).
- Реальные драйверы: **устойчивый edge** (Pinnacle vs PM), **дисциплина лимитов**, **длинная серия** + контроль просадок.
